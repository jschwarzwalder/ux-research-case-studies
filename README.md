# UX Research & Systems Behavior Case Studies

## Overview

This repository contains a collection of UX research, playtesting, and systems behavior case studies spanning multiple domains, including:

- enterprise software ecosystems
- developer experience (DevEx) systems
- VR and interactive environments
- consumer-facing digital products
- educational and public systems
- game and tabletop playtesting environments

The common thread across all work is a focus on understanding how people **interpret, navigate, and break complex systems in real-world conditions**.

## 🧠 Key Insights Across All Work

Across all domains, consistent patterns emerge:

### 1. Users build mental models quickly—even when incorrect
People act on assumptions about systems long before fully understanding them.

### 2. System feedback timing is critical
Delayed or unclear feedback leads to compounding misunderstanding.

### 3. Users attempt to optimize even broken systems
Even when systems are constrained or unsolvable, users continue trying to “solve” them.

### 4. Physical and environmental context shapes comprehension
Embodied or spatial systems significantly change how users learn and interact.

### 5. Observation reveals more than self-reported feedback
Direct behavioral observation is more reliable than post-experience reporting.
